# Report Outline

Section-by-section skeleton with word budgets, targeting ~2000 words total excluding References and
Appendix. These numbers are chosen to sum exactly to the ~2000-word target (see arithmetic below) — if you
adjust one section's budget, adjust another to compensate rather than letting the total drift.

| Section | Target words | Purpose |
|---|---|---|
| Cover Page | n/a | Title, name/ID, module, supervisor, date |
| 1. Executive Summary | 175 | Objectives, datasets, techniques, key results (brief allows 150–200; 175 is the working budget used in the total below) |
| 2. Introduction | 225 | Problem statement, purpose/scope, relevance, metric definitions used, LOs targeted |
| 3. Literature Review (optional) | 100 | Related work, motivation for methods/data — if omitted, redistribute this 100 to Methodology or Results instead of leaving it unused |
| 4. Data Collection & Preprocessing | 275 | Sources, scale, tools + justification, cleaning, merging, challenges |
| 5. Methodology | 275 | Feature engineering, model selection, splitting, algorithmic complexity |
| 6. System Design and Implementation | 275 | Architecture diagram, stack, Spark evidence, security, optional UI |
| 7. Results and Evaluation | 300 | Model metrics, visualisations, system evaluation, validation |
| 8. Critical Reflection | 225 | What worked, challenges, limitations (memory-vs-distributed), future work, ethics |
| 9. Conclusion | 150 | Achievements, LOs met, final thoughts |
| 10. References | n/a (excluded) | APA, all sources/tools |
| 11. Appendices | n/a (excluded) | Code/pseudocode, sample data, extended visuals/logs, UI screenshots, repo link |

**Arithmetic check:** 175 + 225 + 100 + 275 + 275 + 275 + 300 + 225 + 150 = **2,000** words (excluding
References and Appendix), matching the brief's target exactly.

Use `.project/templates/report-section-template.md` per section when drafting.
